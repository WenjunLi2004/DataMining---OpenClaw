大模型对比实验报告

同一份 diagnostic_summary.json 输入三个旗舰模型，对比数据挖掘任务中的分析质量。

1实验概述

将 OpenClaw 流水线生成的结构化事实摘要（diagnostic_summary.json，含 500 个仓库的分类/消融/基线/语言分布等实验数据）作为唯一输入，要求三个大模型撰写包含 7 个固定章节的数据挖掘分析报告，并由自动校验器核查所有引用数字是否与原始事实摘要一致（容差 10%）。

校验机制：OpenClaw 的 insight-analysis skill 会用正则提取模型输出中的所有数字， 逐一核查是否能在 diagnostic_summary.json 中找到对应值（±10% 容差）。 分子/分母 = 未通过校验的数字数 / 总引用数字数，数字越小说明模型引用越准确。

3分析深度对比

第 1 节：实验结论

模型表现
DeepSeek 最详尽：分别给出各模型 AUC/PR-AUC/F1/P@10/P@20，计算了 Python 正例率偏差（+0.416）和模型 vs 基线的 AUC 提升量（+0.225）
Kimi 简洁：4 个要点，突出 RF 最佳、P@10 优势，时间验证结果。省略 P@20 等细节
Qwen 精确：引用了完整的精确值（AUC 0.887922），明确比较了 LR/RF/XGBoost 三者分工

第 3 节：数据集问题诊断

这一节最能体现模型的批判性思维能力。

DeepSeek · 文字分析型

对每个问题展开段落式论证，如："Python 的 per-language AUC 仅 0.674，低于有可比 AUC 的语言平均值 0.130"——主动计算了均值差距，而非直接复述原始数字。

Kimi · 表格结构型

使用"问题 | 证据 | 影响"三列表格呈现，一眼可扫描。节省阅读时间，但分析不如文字版深入。

Qwen · 溯源审计型

主动核查了数据血缘："当前数据声明所有 500 行均使用了严格的 30 天收集器（all_strict_30d: true）"——是三个模型中唯一主动验证数据来源元数据的。

第 6 节：下一步实验建议

三者风格差异最明显：

  • DeepSeek：学术风格，给出 5 条具体可执行的实验方向（含重采样、消融、多切分点验证）
  • Kimi：产品风格，用 P0/P1/P2 优先级分类建议，更像工程 backlog 而非学术实验计划
  • Qwen：合规风格，建议中多次提及"保持可重复性"、"添加免责声明"，侧重规范性

4风格特点总结

DeepSeek V4 Pro

风格：学术论文型

优势：分析最深入，会做二次推导（如计算均值差）；信息密度最高

劣势：有 2 个数字略偏差；篇幅最长，有重复信息

适合场景：需要深度探索性分析，对完整性要求高

Kimi K2

风格：工程决策型

优势：100% 数字准确；表格 + 优先级标注，决策者友好；最简洁

劣势：引用信息量只有 DeepSeek 一半；深度分析不足

适合场景：需要快速决策、给非技术人员看的汇报

Qwen3.7 Max

风格：精确审计型

优势：100% 准确；唯一主动核查数据元数据；引用精确到小数点后 6 位

劣势：格式较模板化，洞察深度居中

适合场景：需要严格可溯源性，审计或合规场景

5各模型独特发现

DeepSeek V4 Pro 独有观察

  • 主动计算了 Python AUC 与"非 Python 语言平均 AUC"之间的差距（0.130），而原始数据中没有这个数字——说明它在做真正的推理，而不只是复述
  • 指出派生特征的冗余："从 C 组到 D 组仅 +0.0013 AUC，特征可减法"
  • 提出了一个有价值的替代标签假设："若将未来标签从 current_stars 换成 360 天后星标增量…"

Kimi K2 独有观察

  • 唯一使用 P0/P1/P2 优先级框架，把学术建议转化为产品迭代计划
  • 明确指出 readme_len_30d 与 commits_30d 的相关系数仅 0.215，说明 README 特征是独立信号——这是对特征间关系的正面论证，其他模型只聚焦高相关问题

Qwen3.7 Max 独有观察

  • 唯一主动说明 SHAP 数据缺失："top10_shap 为空，无法从边际贡献角度验证"——说明它读取了 JSON 结构，而不只是数字
  • 注意到时间切分测试集正例率下降（0.15 vs 全局 0.204），并提出了有价值的假设："近期仓库 star 增长趋势是否整体放缓"
  • 引用精确度最高，AUC 引用为 0.887922 而非四舍五入的 0.888

核心结论：三个模型体现了不同的权衡策略，没有绝对最优。

DeepSeek V4 Pro："宁可多说，不怕偏差"——信息最密集，推理最深，2/146 的偏差率可接受，适合探索性分析。
Kimi K2："只说有把握的"——数字最保守（仅引用 77 个），全部准确，结构最清晰，适合向非技术决策者汇报。
Qwen3.7 Max："既要准确，也要溯源"——居中方案，唯一主动审计数据来源，精确引用，适合需要可审计性的场景。
对 OpenClaw 框架的启示：这一对比验证了 OpenClaw 设计原则的必要性—— 即使是最优秀的大模型（DeepSeek V4 Pro 98.6% 准确率），在 146 次引用中仍有 2 次偏差。 在数据挖掘任务中，先算事实、再让大模型基于事实推理的受约束架构， 能将模型的"编造"风险从不可控降低到可量化的程度。
